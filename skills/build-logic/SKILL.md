# Android Gradle Logic Expert Skill

Expertise in Gradle best practices and managing complex project configurations.

## 🧠 Core Concepts
- **Version Catalogs:** Standardize dependency management across multi-module projects using `libs.versions.toml`.
- **Custom Plugins:** Encapsulate build logic into reusable Gradle convention plugins.
- **Build Optimization:** Improve build speeds and maintain consistency across development environments.

## 🛠 Rules
1. **Convention over Configuration:** Use convention plugins to share build logic between modules.
2. **Type-safe Accessors:** Prefer Kotlin DSL and version catalogs for type-safety.
3. **Lazy Configuration:** Use Gradle properties and providers to avoid eager configuration.
4. **Dependency Hygiene:** Regularly audit dependencies and remove unused ones.
